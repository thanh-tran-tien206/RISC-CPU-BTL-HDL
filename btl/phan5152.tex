\subsubsection{Testbench và Kết quả mô phỏng Instruction Register (IR)}

\textbf{a. Mô tả kịch bản kiểm chứng (Testbench)} \\
Môi trường mô phỏng cho khối Instruction Register được thiết lập nhằm xác minh khả năng phân rã lệnh 32-bit thành mã máy (Opcode) và địa chỉ (Address), đồng thời kiểm tra tính chính xác của cơ chế nạp lệnh có điều kiện. Các kịch bản kiểm thử bao gồm:
\begin{itemize}
    \item Kiểm tra đặc tính \textbf{Reset đồng bộ}: Kích hoạt \texttt{rst} để đảm bảo các trường \texttt{opcode} và \texttt{addr} được đưa về giá trị 0 tại cạnh lên của xung clock.
    \item Kiểm tra \textbf{khả năng giải mã}: Nạp các gói dữ liệu 32-bit khác nhau để xác nhận các bit cao (31-29) được tách vào \texttt{opcode} và các bit thấp (28-0) được tách vào \texttt{addr}.
    \item Kiểm tra \textbf{cơ chế chốt dữ liệu}: Thay đổi ngõ vào \texttt{data\_in} khi tín hiệu \texttt{ld\_ir} ở mức thấp để chứng minh dữ liệu cũ vẫn được duy trì, ngăn chặn việc nạp lệnh sai thời điểm.
\end{itemize}

\textbf{b. Mã nguồn Testbench (Verilog)}
\begin{lstlisting}[language=Verilog, caption={Mã nguồn Testbench kiểm tra Instruction Register}]
`timescale 1ns / 1ps

module tb_instruction_reg();

    reg          clk;
    reg          rst;
    reg          ld_ir;
    reg  [31:0] data_in;
    
    wire [2:0]  opcode;
    wire [28:0] addr;

    // Khoi tao module IR (Unit Under Test)
    instruction_reg uut (
        .clk(clk),
        .rst(rst),
        .ld_ir(ld_ir),
        .data_in(data_in),
        .opcode(opcode),
        .addr(addr)
    );

    // Tao xung clock chu ky 10ns
    always #5 clk = ~clk;

    initial begin
        // Khoi tao gia tri ban dau
        clk = 0; rst = 0; ld_ir = 0; data_in = 32'b0;

        $display("----------");
        $display(" Time|rst|ld_ir|Data_in|Opcode|Address");
        $display("----------");
        $monitor("%10t|%b|%b|%h|%b|%h", 
                 $time, rst, ld_ir, data_in, opcode, addr);

        // 1. Kiem tra Reset dong bo
        #12; rst = 1;
        #10; rst = 0;

        // 2. Thu nap lenh ADD (010) voi dia chi 0x00000FF
        #10; 
        data_in = {3'b010, 29'h00000FF}; 
        ld_ir = 1; // Cho phep nap
        #10;
        ld_ir = 0; // Khoa nap

        // 3. Thay doi data_in nhung ld_ir = 0 (Gia tri cu phai giu nguyen)
        #10;
        data_in = {3'b111, 29'hFFFFFFF}; 
        #10;

        // 4. Nap lenh moi JMP (111)
        #10;
        ld_ir = 1;
        #10;
        ld_ir = 0;

        // 5. Reset giua chung
        #10;
        rst = 1;
        #10;
        rst = 0;

        #20;
        $display("----------");
        $display("Hoan thanh mo phong Instruction Register!");
        $finish;
    end
endmodule
\end{lstlisting}

Dựa trên log mô phỏng, khi tín hiệu \texttt{ld\_ir} được kích hoạt, giá trị \texttt{opcode} và \texttt{addr} ngay lập tức cập nhật dựa trên 3 bit cao và 29 bit thấp của \texttt{data\_in}. Tại các thời điểm \texttt{ld\_ir} bằng 0, dù \texttt{data\_in} có biến động, các giá trị ngõ ra vẫn giữ nguyên, bảo vệ tính toàn vẹn của lệnh đang thực thi.

Quan sát Waveform cho thấy tại thời điểm Reset hoặc nạp lệnh, dữ liệu chỉ thay đổi khi có sự kiện \texttt{posedge clk}. Các chuyển đổi của \texttt{data\_in} diễn ra giữa hai chu kỳ xung nhịp không gây ảnh hưởng đến hoạt động nội tại của CPU nhờ vào cơ chế chốt dữ liệu (latching mechanism) của thanh ghi lệnh.
