\subsubsection{Testbench và Kết quả mô phỏng Address Mux}

\textbf{a. Mô tả kịch bản kiểm chứng (Testbench)} \\
Môi trường mô phỏng cho khối Address Mux được thiết lập nhằm xác minh tính chính xác của logic tổ hợp trong việc lựa chọn nguồn địa chỉ truy xuất bộ nhớ. Các kịch bản kiểm thử tập trung vào:
\begin{itemize}
    \item \textbf{Kiểm tra tính phản hồi tức thời}: Xác nhận ngõ ra \texttt{addr\_out} cập nhật ngay khi tín hiệu chọn \texttt{sel} thay đổi mà không phụ thuộc vào xung nhịp.
    \item \textbf{Mô phỏng chu kỳ máy}: 
        \begin{itemize}
            \item Thiết lập \texttt{sel = 1} để kiểm tra luồng địa chỉ từ Program Counter (giai đoạn Fetch).
            \item Thiết lập \texttt{sel = 0} để kiểm tra luồng địa chỉ từ Instruction Register (giai đoạn Execute).
        \end{itemize}
    \item \textbf{Kiểm tra tính độc lập}: Thay đổi dữ liệu tại ngõ vào không được chọn để đảm bảo ngõ ra vẫn giữ nguyên giá trị từ nguồn đã chọn, minh chứng cho tính cách ly của bộ Mux.
\end{itemize}

\textbf{b. Mã nguồn Testbench (Verilog)}
\begin{lstlisting}[language=Verilog, caption={Ma nguon Testbench kiem tra Address Mux}]
`timescale 1ns / 1ps

module tb_address_mux();

    // Khai bao tham so do rong bit
    parameter W = 32;

    reg  [W-1:0] pc_addr;
    reg  [W-1:0] ir_addr;
    reg          sel;
    wire [W-1:0] addr_out;

    // Khoi tao module Address Mux voi tham so W (Unit Under Test)
    address_mux #(.WIDTH(W)) uut (
        .pc_addr(pc_addr),
        .ir_addr(ir_addr),
        .sel(sel),
        .addr_out(addr_out)
    );

    initial begin
        // In tieu de theo doi
        $display("-----------");
        $display(" Time|sel|pc_addr|ir_addr|addr_out");
        $display("-----------");
        $monitor("%10t|%b|%h|%h|%h", 
                 $time, sel, pc_addr, ir_addr, addr_out);

        // Khoi tao gia tri ban dau
        pc_addr = 32'h0000_1000; // Dia chi lenh (Fetch)
        ir_addr = 32'h0000_ABCD; // Dia chi toan hang (Execute)
        sel = 0;

        // 1. Test chon ir_addr (Execute Phase)
        #10; sel = 0; 
        // 2. Test chon pc_addr (Fetch Phase)
        #10; sel = 1;

        // 3. Thay doi pc_addr khi dang chon pc_addr
        #10; pc_addr = 32'h0000_1004;

        // 4. Thay doi ir_addr khi dang chon pc_addr (addr_out khong doi)
        #10; ir_addr = 32'h0000_EEEE;

        // 5. Chuyen sang chon ir_addr
        #10; sel = 0;

        #10;
        $display("-----------");
        $display("Hoan thanh mo phong Address Mux!");
        $finish;
    end
endmodule
\end{lstlisting}

Dựa trên bảng kết quả, tại thời điểm \texttt{sel = 1}, \texttt{addr\_out} luôn bám sát giá trị của \texttt{pc\_addr}. Khi thay đổi \texttt{ir\_addr} trong khi \texttt{sel} vẫn bằng 1, ngõ ra hoàn toàn không bị ảnh hưởng, đảm bảo tính ổn định của địa chỉ truy xuất trong giai đoạn Fetch.

Điều này giúp loại bỏ độ trễ chu kỳ xung nhịp trong việc xác định địa chỉ bộ nhớ, tối ưu hóa tốc độ truy xuất dữ liệu của CPU.
